# 祖玛游戏 — 遇到的问题与修复记录

## 问题 1：消除后得分不增加

### 现象
射击消除珠子后，分数始终为 0，特效也没有触发。

### 原因
`chain.js` 中 `_checkEliminate()` 返回了消除结果，但 `main.js` 中没有正确读取并处理该结果。最初代码只是简单判断了 `hitResult` 存在，没有实际加分。

### 修复
1. 在 `Chain` 类中添加 `lastEliminateResult` 属性，消除时记录：
   - 消除球数 (count)
   - combo 层数 (combo)
   - 消除位置 (x, y)
   - 颜色 (color)
2. 在 `main.js` 的 `update()` 中每帧检查 `chain.lastEliminateResult`，读取后加分、触发特效，并置为 null。
3. 这样不仅射击消除能加分，连锁消除（merge 接合触发的）也能正确加分。

### 关键代码
```javascript
// chain.js - _checkEliminate() 中
this.lastEliminateResult = {
  count: matchCount,
  combo: this.comboCount,
  x: centerBall.x,
  y: centerBall.y,
  color: centerBall.color.fill,
};

// main.js - update() 中
if (chain.lastEliminateResult) {
  const { count, combo, x, y, color } = chain.lastEliminateResult;
  const points = count * 10 * combo;
  score += points;
  effects.addEliminateEffect(x, y, color);
  effects.addScorePopup(x, y, points, combo);
  chain.lastEliminateResult = null;
}
```

## 问题 2：连锁消除可能丢失事件

### 现象
快速连锁消除时（merge 接合连续触发多次消除），只有最后一次消除的得分被记录。

### 原因
`lastEliminateResult` 是单值，如果一帧内多次消除会被覆盖。

### 状态
**已知问题，暂未修复。** 当前方案在大多数情况下可正常工作（merge 通常跨多帧完成），极端情况下可能丢失中间消除的得分。

### 未来改进方案
将 `lastEliminateResult` 改为数组队列 `eliminateResults[]`，main 中每帧清空并逐个处理。

## 问题 3：弧长参数化精度

### 现象
贝塞尔曲线的 `t` 参数不是等距的，直接用 `t` 会导致珠子在曲线弯曲处间距不均匀。

### 解决方案
使用弧长参数化：
1. 预采样 1000 个点
2. 计算累计距离数组 `distances[]`
3. 给定弧长 `d`，通过二分查找找到对应的采样区间
4. 在区间内线性插值得到精确位置

这确保了珠子无论在直线段还是弯曲段都保持等距。

## 问题 4：插入后推挤可能越界

### 现象
在轨道末端附近插入球时，推开后方球可能导致 dist 超出轨道总长度。

### 当前处理
`path.getPointAtDistance()` 中对 dist 做了 clamp：
```javascript
d = Math.max(0, Math.min(this.totalLength, d));
```
超出部分的球会堆积在终点位置，下一帧触发失败判定。

### 状态
**已处理**，行为符合预期（球到达终点 = 游戏失败）。

## 问题 5：Canvas 尺寸适配

### 现象
不同屏幕尺寸下游戏可能显示不全或太小。

### 解决方案
```javascript
function resizeCanvas() {
  canvas.width = Math.min(window.innerWidth, 900);
  canvas.height = Math.min(window.innerHeight, 650);
}
```
- 限制最大为 900×650
- 关卡轨道控制点使用 `w` 和 `h` 的百分比定义，自动适配
- 监听 `resize` 事件动态调整

## 问题 6：deltaTime 过大导致穿透

### 现象
切换标签页再切回时，`dt` 可能非常大（几秒），导致子弹直接飞过珠子链。

### 解决方案
限制最大 dt：
```javascript
const dt = Math.min((timestamp - lastTime) / 1000, 0.05);
```
最大 50ms 一帧，防止物理模拟跳跃。

## 问题 7：消除后珠子链断裂散开（关键 Bug）

### 现象
射击消除珠子后，链上剩余珠子加速散开，间距越来越大，链完全断裂无法游戏。

### 定位过程

1. **观察现象**：消除后后段球加速远离前段 → 怀疑 merge 逻辑方向错误
2. **阅读代码**：`_processMerges` 中后段用 `dist += mergeSpeed * dt` → 实际是在增大距离
3. **确认**：消除产生间隙后，后段应该**向前回退（dist 减小）**靠近前段

### 根本原因（三个问题叠加）

**原因 1：merge 移动方向反了**
```javascript
// ❌ 错误：dist 增加 = 远离前段
this.balls[i].dist += mergeSpeed * dt;
// ✅ 正确：dist 减少 = 靠近前段
this.balls[i].dist -= advance;
```

**原因 2：`_enforceSpacing` 条件缺陷**
```javascript
// ❌ 错误：gap < 0.1*BALL_DIAMETER 时不修正
if (gap < BALL_DIAMETER && gap > BALL_DIAMETER * 0.1)
// ✅ 正确：只要 gap < BALL_DIAMETER 就修正（仅限同段内）
if (gap < BALL_DIAMETER)
```
原来跨段间隙和段内间距混在一起处理，现在改为只处理同一段内的球。

**原因 3：`mergeGaps.backStart` 索引失效**
splice 删除球后数组索引变化，但 `backStart` 没有更新。导致：
- 错误的段被当做"后段"而停止正常移动
- `_processMerges` 访问错误的球

### 修复方案

1. **方向修正**：`dist -= advance`，且限制移动量不超过实际间距
2. **间距修正**：按段（segment）内逐个修正，不跨段
3. **索引追踪**：用球 ID (`backBallId`) 追踪后段首球，每次使用前通过 `findIndex` 修正索引

## 附录：日志系统使用说明

### 开启/关闭
在浏览器控制台 (F12) 输入：
```javascript
GameLog.enable()    // 开启日志
GameLog.disable()   // 关闭日志
```

### 调整日志级别
```javascript
GameLog.level = 'debug'   // 全部日志（含珠子生成、回退细节）
GameLog.level = 'info'    // 默认，显示关键事件（消除、插入、得分）
GameLog.level = 'warn'    // 只看警告（异常间距等）
GameLog.level = 'error'   // 只看错误
```

### 过滤模块
```javascript
GameLog.filter = 'chain'    // 只看珠子链日志
GameLog.filter = 'merge'    // 只看接合/回退日志
GameLog.filter = 'bullet'   // 只看子弹日志
GameLog.filter = 'shooter'  // 只看射手日志
GameLog.filter = 'game'     // 只看游戏状态日志
GameLog.filter = null       // 全部模块
```

### 查看历史
```javascript
GameLog.dump()    // 打印最近 200 条日志（含关闭时的记录）
GameLog.clear()   // 清空历史
```

### 日志覆盖点

| 模块 | 级别 | 内容 |
|------|------|------|
| chain | debug | 珠子生成 |
| chain | info | 子弹命中、珠子插入、消除 |
| chain | warn | 球间距异常（负值） |
| merge | info | 链段接合、创建 merge gap |
| merge | warn | gap 索引失效 |
| merge | debug | 后段回退进度 |
| bullet | debug | 发射、飞出画布 |
| shooter | debug | 射击、换球 |
| game | info | 关卡初始化、得分、胜利、失败 |

> 注意：即使日志关闭，事件仍会写入内存历史，可通过 `GameLog.dump()` 随时查看。
